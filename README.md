# Graph Alignment: FUGAL vs FAQ Benchmarking

This repository contains the experimental code for benchmarking [FUGAL](https://dl.acm.org/doi/10.5555/3737916.3738532) against [FAQ](https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0121002) on real-world graphs under noise, with a focus on the effect of warm initialization and hyperparameter tuning.

## Overview

**Graph alignment** (a.k.a. network alignment) is the problem of finding a node-to-node correspondence between two graphs that best preserves edge structure. Given adjacency matrices `A` and `B`, the goal is to find a permutation matrix `P` minimizing `||A - P B P^T||_F`.

This project investigates:
- FUGAL vs FAQ under **uniform initialization**
- FUGAL vs FAQ under **warm initialization** (Frank-Wolfe seed via `relaxed_normAPPB_FW_seeds`)
- Effect of FUGAL's hyperparameters `mu` (feature regularization weight) and `lam_step` (doubly-stochastic penalty step size) on alignment quality
- A improved alpha (step-size) schedule for FUGAL borrowed from FAQ's Frank-Wolfe line search

## Algorithms

| Name | Description |
|------|-------------|
| `fugal` | FUGAL with uniform initialization |
| `fugal_init` | FUGAL with FW warm initialization |
| `qap` | FAQ (`scipy.optimize.quadratic_assignment`) with uniform init |
| `qap_init` | FAQ with FW warm initialization |

The key FUGAL variant used in final experiments is `convex_init_complex_alpha` in `src/GA_Archana.py`, which combines FUGAL's Sinkhorn-based doubly-stochastic relaxation with FAQ's quadratic step-size line search and a graduated penalty schedule.

## Running Experiments

The orchestrator runs in two phases automatically:

**Phase 1** generates noisy graph pairs and FW seed matrices and caches them to disk (`baseline_data/`, `baseline_fw_seeds/`).

**Phase 2** runs the chosen algorithm(s) and appends results to a timestamped CSV in `results/`.

```bash
python tests/orchestrator.py \
    --algos fugal_init qap_init \
    --ds netscience highschool euroroad multimanga voles \
    --mu 0.5 \
    --lam-step 0.4
```

### Arguments

| Argument | Description | Default |
|----------|-------------|---------|
| `--algos` | Algorithms to run: `fugal`, `fugal_init`, `qap`, `qap_init` | `qap_init` |
| `--ds` | Datasets to use | `netscience` |
| `--mu` | Feature regularization weight(s) for FUGAL | `0` |
| `--lam-step` | Penalty step size(s) for FUGAL | `1` |
| `--dry-run` | Print config without running | off |

Noise levels (0%, 5%, 10%, 15%, 20%) and trial seeds (42, 123, 7) are fixed in the script.

### Output

Each run produces two files in `results/`:
- `exp_<timestamp>.csv` — per-run rows with columns: `dataset, noise, trial, algorithm, mu, lam_step, accuracy, frobenius, gt_frobenius`
- `summary_<timestamp>.txt` — timestamped log of all print output

## Metrics

- **Accuracy**: fraction of nodes correctly aligned (compared to ground-truth permutation)
- **Frobenius norm**: `||A - P B P^T||_F^2` for the predicted permutation `P`
- **GT Frobenius norm**: same metric using the ground-truth permutation (lower bound)

## Noise Model

For each dataset, noisy graph pairs are generated by:
1. Randomly permuting node labels to get the ground-truth mapping
2. Removing edges independently with probability `noise` (preserving connectivity)

Three independent trials per noise level are averaged.
